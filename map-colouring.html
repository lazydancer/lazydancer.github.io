Map Colouring

Apr 2019

For the second assigment of the discrete opimization course, you need to colour a map where each neighbouring country has to be a different colour. The challenge is to find the smallest number of colours where you can colour every country. Like the other problems so far in this course this is an NP problem.

This in an challenge in which the difficulty grows exponentally with the number of countries added. We can generalize the map and change it into a graph. This allows us to have as many bordering counties as we want. I will call countries “nodes” as they are represented in a graph. This type of problem is also called vertex colouring.

Brute force

I am going to start by trying every single combination and returning the best solution. Computers are great with numbers and I want to be lazy.

The computer 2 million combinations a second. With a 2GHz computer that is a check every 1000 cycles. I’m sure we could get 10x or 100x optimizing the code but I’ll show you below, we need much more than that.

The number of possible combinations is equal to number of colors ^ number of nodes.

Example: (3 colours and 20 nodes)

3 ^ 20 = 3,486,784,401 or 3.4 * 10^9 possibilities

Which would take our brute force 2 seconds to complete

World Map: (4 colours, 195 nodes)

4 ^ 195 = 2.5 * 10^117 possibilities

Which would take our brute force 10^111 seconds or 10^103 years. We need another way.

Pick most connected

Instead of trying every combination, lets order the way we select the nodes we colour. Starting from the node which has the most connections, choosing its colour and moving to the next adding a new colour if neighbouring nodes used all the previous colors. This will go on till all the colors are choosen

This is so much faster! It is a greedy algoritym so it doesn’t give us an optimal result but the speed is important here. When you add a new node the time only increases a bit each time and is closer to linear.

Saturation

This method is very similar to the most connected but in this case I will update each time and pick the one that has been chosen updating it’s saturation, how many differnent colors does it neighbour with. The other name for this algorithm is DSATUR and is a well known algoritym for completing this problem

The first thing I do is create adjantency matrix.

From there we can use the following equation and if an element in the matrix is true then 2 nodes are the same colour and that is not a solutoin

adjMatrix * sol and adjMatrix * transpose(sol)

II believe it is because the improved brute force is only checking the connections. Where the matrix will be checking every possible connection, even if the connection doesn’t exist it will need to do 0 * sol == 0 * Transpose(sol). In addition the original brute will fail in the first case it sees a break in the conditions and the first has to calculate to the end.
